# Person-Practice 项目配置

## PPT 制作规范

使用 python-pptx 生成 PPT 时，必须遵守以下规则：

### 画布尺寸
- **16:9 比例**：10 × 5.625 英寸（25.4 × 14.2875 cm）
- 所有元素的 `top + height` 必须 ≤ 5.625 英寸
- 所有元素的 `left + width` 必须 ≤ 10 英寸
- 建议保留 0.2~0.3 英寸安全边距

### 排版约束
- 每页内容不要塞太满，留出呼吸空间
- 文字框高度要预留足够空间，避免文字溢出被截断
- 圆角矩形卡片底部不要贴着画布底部
- 多列布局时，列与列之间保持 0.3 英寸间距

### 配色主题（奶油色浅色系）
- 背景：`#FAF7F2`（奶油白）
- 主色：`#2D5B7B`（深蓝）
- 辅色：`#4A90D9`（亮蓝）
- 强调：`#E86D50`（珊瑚橙）
- 浅色背景：`#E3F2FD`（蓝）、`#E8F5E9`（绿）、`#FFF3E0`（橙）、`#F3E5F5`（紫）

### 文件输出
- 输出目录：`D:/pycharm/Person-Practice/PPT/`
- 虚拟环境：`D:/pycharm/Person-Practice/.venv/Scripts/python.exe`
- print 语句不要用 emoji（Windows GBK 编码会报错）

### 画布尺寸（宽屏版）
- **16:9 宽屏**：13.333 × 7.5 英寸（MckEngine 默认）
- 与标准 16:9（10×5.625）不同，内容区更大

## PPT 制作经验（踩坑总结）

### MckEngine 的局限性
- MckEngine 是麦肯锡咨询风格，设计语言固定（商务极简），不适合做美术设计风格的 PPT
- 如果要做有色彩层次、东方美学等风格，**不要用 MckEngine，直接用 python-pptx 从零写**

### 颜色覆盖的坑（Python 模块机制）
- `from module import *` 在 import 时就复制了值，后续改模块属性**不会生效**
- 修改 MckEngine 颜色的唯一方式：直接改 `mck_ppt.engine` 模块的命名空间属性（`E.NAVY = X`），但仍然只对直接引用 `E.NAVY` 的代码有效
- **结论：想完全控制颜色，就不要用 MckEngine，从零写**

### Theme 颜色 vs 内联颜色
- PPT 的颜色有两个层级：**theme 层**（`ppt/theme/theme1.xml` 里的 `clrScheme`）和**内联层**（每个 shape 的 `srgbClr`）
- MckEngine 的 shape 很多引用 theme 色（`accent1`、`dk2` 等），只改内联颜色**看不到变化**
- 要改 theme 需要直接操作 XML（zip 解压 → 改 theme1.xml → 重新打包）

### 正确的制作流程
1. **内容 & 主题** → 先定内容大纲、受众、页数
2. **静态排版** → python-pptx 从零写，每个 shape 的颜色/坐标/大小都直接控制
3. **动态动画** → win32com 在静态版基础上添加（已验证可用）

### 设计原则
- **大面积用主色**，不要只在小圆点上出现 — 用户要看到颜色的冲击力
- **两种主色要交融**，不是分开放 — 如石榴裙红做色块 + 明月珰珠光做底色
- **对称布局配色互换** — 左右对比时，布局一致但颜色相反，视觉效果好
- **每页不要留大面积空白** — 用色块、卡片、分隔线填满内容区

### 动画制作（win32com）
- **脚本位置**：`D:/pycharm/Person-Practice/PPT/add_animations.py`
- **原理**：打开静态 PPT → 逐页逐 shape 添加动画 → 另存为新文件
- **动画方案**：每页第一个形状 ON_CLICK（点击触发），后续 AFTER_PREVIOUS（自动跟上，0.1s 延迟）
- **淡入时长**：0.5s
- **页面切换**：PP_FADE（平滑淡出），1.0s
- **注意**：需要 PowerPoint 已安装且可启动（COM 自动化）
- **输出文件名**：在原文件名后加 `_动画` 后缀
